# 视频分析后端接入

## 真实链路

1. 前端选择 MP4 / MOV 视频并校验大小。
2. 本地开发通过 multipart 提交；Vercel 生产通过 `@vercel/blob/client` 直传，避免 Function 4.5MB 请求体限制。
3. `POST /api/analysis-jobs` 创建任务并立即返回 `jobId`。
4. 第一次调用 `qwen3.7-plus` 只抽取视频事实、动作 ID 和关键帧时间。
5. 本地政策证据库按六维返回证据卡片。
6. 第二次调用只接收视频事实、宝宝档案和证据卡片，生成“宝宝版本 / 视频解析 / 陪做步骤”。
7. Zod 校验 5 个关键判断、6 个维度、唯一动作 ID，以及原视频时间戳不可改写。
8. FFmpeg 按模型返回的 `keyframe_time` 截图；截图失败不影响文本结果。
9. 前端轮询任务，完成后展示结果并进入对话陪做。

## 环境变量

本地使用 `.env.local`，不要提交真实密钥。部署到 Vercel 时配置：

```text
DASHSCOPE_API_KEY=
QWEN_MODEL=qwen3.7-plus
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
BLOB_READ_WRITE_TOKEN=
TIKHUB_API_KEY=
TIKHUB_BASE_URL=https://api.tikhub.io
```

`BLOB_READ_WRITE_TOKEN` 在 Vercel 项目中连接 Blob Store 后自动生成。没有该变量时，本地任务、视频和关键帧保存在被 Git 忽略的 `tmp/analysis-jobs/`。

## 能力边界

- 抖音分享链接由服务端优先通过 TikHub App V3 解析为临时视频地址，空结果按官方建议回退 Web 接口；TikTok 使用 Hybrid Parsing。API Key 不会发送到浏览器；其他 HTTPS 地址按视频直链处理。当前不支持小红书分享页。
- TikHub 默认使用海外域名 `api.tikhub.io`；只有后端部署在中国大陆时才将 `TIKHUB_BASE_URL` 改为 `https://api.tikhub.dev`。
- 只处理公开且用户有权使用的内容；私密、已删除、地区受限内容不会尝试绕过权限。
- “待确认”表示输入不足，不等于过敏或禁忌。
- 只有“可以直接做”和“调整后可以做”可以进入陪做；其他结论必须补资料或更换视频。
- 当前政策规则适用于健康足月、满 6 月龄至 24 月龄宝宝；早产或特殊医学状况只作参考并提示寻求专业意见。
- Vercel 生产必须配置 Blob；不能用内存 Map 保存任务，也不能让 200MB 视频经过 Function 请求体。
