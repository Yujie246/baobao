# 宝宝饱饱首页信息架构与优化方案

## 1. 首页目标

首页不是功能入口集合，而是宝宝每天的辅食工作台。它优先回答三个问题：

1. 我现在最应该做什么？
2. 今天可以参考什么辅食？
3. 宝宝最近吃得怎么样？

所有展示必须来自真实档案、计划、制作记录或观察记录；没有数据时展示空状态，不用固定 Mock 冒充进展。

## 2. 核心设计原则

- **任务优先**：存在进行中制作或待观察任务时，任务卡排在内容导入之前。
- **状态驱动**：首页顺序随用户当前状态变化，不固定堆放七个模块。
- **能力真实**：没有完成上传、解析和失败恢复前，不展示可用的上传入口。
- **避免安全承诺**：使用“今日辅食灵感”，不使用容易被理解为安全判断的“今天可以吃什么”。
- **首屏有效**：首屏至少露出 2.5 个模块，每个模块都提供状态、推荐依据或下一步操作。
- **渐进披露**：首页只展示摘要，食物分类、完整计划和详细记录进入二级页面查看。

## 3. 动态页面顺序

### 3.1 有进行中任务

1. 顶部宝宝信息
2. 下一步任务
3. 导入辅食内容
4. 今日辅食灵感
5. 本周辅食计划与食物探索地图
6. 本周进展

### 3.2 没有进行中任务

1. 顶部宝宝信息
2. 导入辅食内容
3. 今日辅食灵感
4. 本周辅食计划与食物探索地图
5. 本周进展
6. 空闲状态提示

## 4. 模块方案

### 4.1 顶部宝宝信息

展示内容：

- 品牌小字：宝宝饱饱
- 主标题：今天给满满做什么？
- 信息胶囊：满满 · 10个月 · 软颗粒
- 右上角：宝宝 IP 头像，点击进入宝宝档案

顶部保持紧凑，不增加长段辅助文案。

### 4.2 下一步任务

首页只突出当前优先级最高的一件事，优先级为：

1. 异常待处理
2. 正在制作
3. 已到时间、需要观察
4. 稍后需要观察
5. 暂无任务

进行中制作示例：

- 标题：继续制作
- 菜名：宝宝虾滑面
- 状态：做到第 1 步
- 时间：预计还需 12 分钟
- 操作：继续陪做

如果还有其他任务，在主任务下提示“还有 N 项待处理”，不并列堆放多张大卡。

### 4.3 导入辅食内容

目标是让用户快速把外部内容转成宝宝版本。

入口规划：

- 粘贴链接
- 上传视频
- 上传图文

当前只开放已经真实完成的入口。上传功能开放前必须具备：文件校验、上传进度、失败恢复、重新选择、服务端解析和清理机制。

粘贴状态：

- 输入框：粘贴辅食视频链接
- 次操作：粘贴
- 主操作：开始分析
- 示例入口：宝宝虾滑面、香蕉燕麦饼、南瓜蒸糕

上传状态未来在同一卡片内切换为预览，不额外增加页面模块。

### 4.4 今日辅食灵感

标题使用“今日辅食灵感”，避免形成食物安全保证。

推荐卡采用横向滑动，露出约 2.3 张。每张包含：

- 食物小伙伴插画
- 菜名
- 制作时间
- 推荐依据
- 查看宝宝版本

推荐依据必须可追溯，例如：

- 当前质地阶段匹配
- 主要食材已有尝试记录
- 本周尚未重复
- 制作时间较短

“换一换”只改变当前候选集合，不制造没有依据的随机推荐。

### 4.5 本周计划与食物地图

两个模块使用并排摘要卡。

本周辅食计划：

- 已安排天数
- 今天的主要安排
- 查看计划

食物探索地图：

- 已记录食材数
- 本周新增数
- 继续探索

分类统计和最近尝试食材进入地图页查看，首页不重复展示。

### 4.6 本周进展

使用一条横向成长条，展示三个真实指标：

- 新尝试食物数量
- 连续记录天数
- 完成观察的食材

无记录时展示：

> 这周还没有新记录，从第一顿开始积累吧。

## 5. 数据与边界

| 首页信息 | 数据来源 | 无数据处理 |
| --- | --- | --- |
| 宝宝姓名、月龄、阶段 | 宝宝档案 | 引导完成档案 |
| 继续制作 | 本地制作会话 | 不展示任务卡 |
| 待观察任务 | 完成记录与观察时间 | 展示空闲状态 |
| 今日灵感 | 档案、食材记录、计划、推荐规则 | 展示示例并明确标注 |
| 本周计划 | 计划数据 | 显示“还没有安排” |
| 食物数量 | 食材尝试记录 | 显示 0，不使用默认数字 |
| 本周进展 | 历史记录计算 | 展示真实空状态 |

产品不对食物安全、过敏风险或医学适宜性作保证。异常情况应停止尝试并寻求专业帮助。

## 6. 视觉结构

首页使用四类组件建立层级：

1. 大导入卡：核心操作，信息最完整。
2. 横向推荐卡：食物插画为主，适合连续浏览。
3. 状态摘要卡：计划和地图并排。
4. 任务卡／提醒条：强调下一步和进度。

功能入口使用线性图标；推荐、成长和空状态使用食物小伙伴与宝宝 IP。配色延续奶油黄、米粉色、米白和低饱和浅绿。

## 7. 实施优先级

### P0：建立可信首页

- 按任务状态调整首页顺序
- 合并“继续制作”和“观察提醒”为下一步任务
- 完善链接导入卡
- 计划与地图摘要接入现有数据

### P1：增加每日使用价值

- 今日辅食灵感
- 推荐依据展示
- 换一换候选逻辑
- 本周进展计算和空状态

### P2：扩展内容输入方式

- 视频上传
- 图文上传
- 上传进度与失败恢复
- 服务端文件解析与清理

## 8. 验收标准

- 有制作任务时，首屏能直接继续制作。
- 没有任务时，首屏能直接导入内容。
- 首页数字均可追溯到真实数据或明确标注的示例。
- 不出现尚未实现但看起来可以使用的入口。
- 首页推荐不使用安全承诺性文案。
- 三个主导航页面切换时，悬浮底栏保持稳定。
- 小屏设备首屏至少能看到顶部、主任务／导入卡和下一模块的一部分。
