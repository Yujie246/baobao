# 宝宝饱饱：角色视觉规范

> 状态：全站扩展方案稿
> 适用范围：宝宝档案、首页探索、计划、食物地图、视频分析、适配结果、对话陪做、反馈与记录
> 参考图：`/Users/bytedance/Downloads/ChatGPT Image 2026年7月22日 11_06_09.png`
> 更新日期：2026-07-22

## 1. 目标

以参考图中的黄色小鸡为宝宝饱饱的角色形象，在全产品中承担四类作用：

1. 对用户刚刚选择的年龄、忌口和进食能力给出即时视觉反馈；
2. 在首页、计划和食物地图中引导用户理解“接下来可以做什么”；
3. 在视频分析、结果和陪做流程中成为稳定的产品向导与对话者；
4. 在完成、反馈、空状态和错误恢复时提供情绪承接。

角色插图可以是状态反馈、流程向导或品牌陪伴，但不是新的输入控件，也不替代文字、校验、风险图标和无障碍信息。

## 2. 使用边界

### 2.1 高频使用

- `/onboarding/age`：根据实际月龄切换年龄形象；
- `/onboarding/avoid`：根据忌口回答状态切换动作；
- `/onboarding/stage`：展示四种进食阶段及食物质地；
- `/home`、`/plan`、`/food-map`：角色承担首页入口、计划和探索引导；
- `/analysis/:id`、`/result/:conclusion`：角色承担读取、检查、解释和等待补充信息；
- `/cook/:id/session`：角色作为 AI 对话头像，并随当前行动切换厨师动作；
- `/cook/:id/complete`、`/feedback/:id/now`：角色承接完成和记录反馈；
- `/baby`、`/baby/edit`：复用已生成的年龄与质地形象。

### 2.2 轻量使用

- `/plan/week`、`/food-map/:food`：只在页头或摘要区使用一张辅助插图；
- `/history`、`/history/:id`：只在页头、完成摘要或空状态使用，不在每条记录重复；
- `/baby/foods`：只在搜索无结果或空状态使用；
- `/analysis-error`、`404`：使用疑问或指路姿势帮助恢复。

### 2.3 不显示完整角色

- 设置与清除数据确认；
- 任何异常、疑似过敏、吞咽困难和紧急医疗提示卡内部；
- 食材长列表的每一行、历史记录的每一条、结果页的每个信息卡；
- 固定底部操作栏、输入框和主按钮内部。

这些区域的第一目标是清楚、可信和快速操作。角色可以出现在页面其他位置，但必须让位于风险信息、真实食物状态和当前操作。

### 2.4 产品边界

- 年龄形象只表达成长感，不表达宝宝理应具备的进食能力；
- 进食能力仍以用户选择的实际表现为准，不根据年龄插图自动推断；
- 纠正月龄不改变基础年龄形象，只在界面中显示纠正月龄标签；
- 插图不构成喂养、发育或医疗建议；
- 风险状态使用现有语义颜色、文字和图标，不把小鸡做成夸张痛苦表情。

## 3. 角色身份

所有状态必须看起来是同一只小鸡，而不是同一风格下的不同角色。

### 3.1 不可变化特征

- 圆润、短胖、近似椭圆的黄色身体；
- 头顶橙色爱心形软冠；
- 白色波浪边围兜；
- 围兜中央黄色爱心胸章；
- 深棕色圆眼睛和高光点；
- 橘粉色圆腮红；
- 小嘴、小短翅膀和短脚；
- 头部与身体之间没有明显细脖子；
- 正面或轻微四分之三视角，不使用强透视。

### 3.2 允许变化特征

- 姿势、手势和视线方向；
- 表情中的微笑、专注、思考和轻微疑问；
- 与当前状态直接相关的道具；
- 随年龄区间轻微变化的身高比例和动作主动程度；
- 厨师帽只用于明确的做饭场景，不作为常驻身份部件。

### 3.3 禁止变化

- 不改变头冠、围兜、胸章和主色；
- 不增加人类头发、鞋、复杂服装或大面积配饰；
- 不出现拟真羽毛、尖锐爪子或真实家禽结构；
- 不出现文字、数字、品牌标志、水印和 UI 按钮；
- 不使用恐怖、疼痛、哭泣或明显医疗化表情；
- 不让道具遮住头冠、眼睛、围兜爱心等身份锚点。

## 4. 视觉 DNA

### 4.1 色彩

| 用途 | 建议色值 | 说明 |
|---|---:|---|
| 身体主色 | `#FFD86A` | 温暖明亮的蛋黄色，避免高荧光 |
| 身体阴影 | `#F1B94C` | 只用于边缘和体积转折 |
| 头冠 | `#F58A3C` | 与参考图一致的橙色强调 |
| 腮红 | `#F29A7E` | 柔和橘粉，不做高饱和红色 |
| 围兜 | `#FFF9EA` | 接近奶油白，保留暖感 |
| 眼睛 | `#3B2717` | 深暖棕，避免纯黑 |
| 线条 | `#C98532` | 低对比暖棕橙轮廓 |
| 页面承载底色 | `#F8F4EC` | 由网页渲染，不进入图片背景 |

### 4.2 材质与画法

- 儿童绘本式暖色水彩与轻蜡笔质感；
- 轮廓柔和但清楚，缩小后仍能识别；
- 体积依赖柔和明暗，不使用硬质 3D 塑料高光；
- 纹理只保留在角色身体、腮红和道具表面；
- 透明背景，不把参考图中的奶油色画布烘焙进素材；
- 不保留地面投影，避免透明图在卡片中出现灰边。

### 4.3 构图

- 角色主体占画布高度约 `78%—84%`；
- 四周保留不少于画布宽度 `8%` 的安全边距；
- 道具与角色形成单一清楚轮廓，不在四周散落装饰物；
- 年龄形象以全身为主；
- 质地形象以半身鸡仔和放大的碗、勺或餐盘为主；
- 小尺寸素材优先确保头冠、眼睛、围兜和食物状态可识别。

## 5. 年龄状态

不为 `4—36` 的每个月生成独立图片，而使用六个连续区间。六个现有快捷月龄会分别命中一个不同状态。

| 区间 | 文件名 | 动作 | 不能表达的含义 |
|---|---|---|---|
| `4—6 月` | `age-04-06.webp` | 体型最小，安静坐姿，双翅放在围兜前 | 不能暗示只会吃泥糊 |
| `7—8 月` | `age-07-08.webp` | 好奇观察，身体轻微前倾 | 不能暗示已经会独坐 |
| `9—10 月` | `age-09-10.webp` | 自然站立，双翅轻扶围兜 | 不能暗示已经会站立 |
| `11—12 月` | `age-11-12.webp` | 开心挥手，另一只翅膀自然下垂 | 不能暗示已经会走路 |
| `13—18 月` | `age-13-18.webp` | 抱着小星星，动作更主动 | 不能暗示认知或语言水平 |
| `19—36 月` | `age-19-36.webp` | 自信的小帮手姿态，双手叉腰或轻挥手 | 不能暗示会独立做饭 |

动作只用于制造视觉差异。年龄段不能用餐具、食物或复杂运动能力区分，以免把年龄和进食能力错误绑定。

### 5.1 映射规则

```ts
function getAgeIllustration(months: number) {
  if (months <= 6) return "age-04-06";
  if (months <= 8) return "age-07-08";
  if (months <= 10) return "age-09-10";
  if (months <= 12) return "age-11-12";
  if (months <= 18) return "age-13-18";
  return "age-19-36";
}
```

输入无效或尚未填写时，显示标准角色母版，不显示某个具体年龄段。

## 6. 忌口状态

| 状态 | 文件名 | 动作与道具 |
|---|---|---|
| 尚未回答 | `avoid-unanswered.webp` | 拿放大镜检查，表情专注但不焦虑 |
| 目前没有 | `avoid-none.webp` | 举绿色勾牌，轻松微笑 |
| 有，需要填写 | `avoid-has-items.webp` | 抱着简单食材清单或夹板，认真确认 |

忌口图只反映回答状态，不把某种具体食材画成“危险食物”。已经选择的具体食材继续由网页标签展示。

## 7. 进食阶段与食物质地

第三步的插图重点是食物状态，不是把四只鸡仔做成能力等级奖章。

| 状态 | 文件名 | 画面要求 |
|---|---|---|
| 刚开始吃辅食 | `texture-puree.webp` | 鸡仔拿小勺，碗里为完全顺滑泥糊；表面连续，没有颗粒 |
| 稠泥和细碎食物 | `texture-thick-puree.webp` | 较稠泥糊能在勺上保持形状，只有极少量细碎点 |
| 压软的小颗粒 | `texture-soft-lumps.webp` | 碗内有明确、柔软的小颗粒；鸡仔用勺背轻压展示可压软 |
| 软手指食物 | `texture-finger-food.webp` | 盘中是容易压软的短条状食物；鸡仔轻拿一条，不出现硬脆食材 |

### 7.1 小尺寸要求

- 每张卡片左侧显示约 `56×56px` 插图；
- 食物区域至少占图片有效面积的 `35%`；
- 缩小到 `48×48px` 时，仍能区分顺滑、稠泥、软颗粒和条状食物；
- 不依赖细小纹理或文字说明完成区分；
- 选中态由网页卡片的暖黄色底色和单选状态表达，图片自身不使用红绿边框。

## 8. 页面接入

### 8.1 年龄页

- 将现有 `52×52px` 问题图标替换为约 `132×132px` 的动态年龄插图；
- 插图放在问题标题上方，作为本页唯一主视觉；
- 月龄变化后立即切换对应图片；
- 纠正月龄只显示文字标签，不更换主图；
- 固定底部主按钮规则保持不变。

### 8.2 忌口页

- 使用约 `104×104px` 的状态插图；
- 未回答、目前没有和有忌口三种状态切换图片；
- 已选食材标签和清空确认逻辑保持不变；
- 插图不能替代“没吃过不等于不能吃”的文字说明。

### 8.3 进食能力页

- 不增加独立大幅头图，避免页面过长；
- 用四张质地插图替换当前选项卡中的数字方块；
- 选项文字、最近进食情况和补充输入保持原逻辑；
- 吞咽费力等风险提示仍使用语义色和盾牌图标，不使用角色图。

## 9. 动效

- 状态切换：旧图淡出，新图淡入并从 `scale(0.97)` 过渡到 `scale(1)`；
- 时长：`220ms`；
- 缓动：`cubic-bezier(0.22, 1, 0.36, 1)`；
- 图片加载完成后才开始切换，避免空白闪烁；
- 年龄页预加载当前和相邻区间素材；
- `prefers-reduced-motion: reduce` 时取消缩放，仅做即时替换；
- 不制作 GIF、视频、逐帧动画、Canvas 或 WebGL 效果。

动效只确认状态变化，不持续吸引注意。

### 10.1 输出规格

- 生成源文件：`1024×1024 PNG`；
- 背景：透明；
- 网页运行素材：透明 `WebP`；
- 年龄和忌口素材：建议输出 `320×320` 与 `640×640` 两档；
- 质地卡片素材：建议输出 `160×160` 与 `320×320` 两档；
- 单张运行素材目标：不超过 `80KB`；
- 所有素材必须使用相同画布、角色比例和安全边距；
- 保留 PNG 母版，WebP 仅作发布版本。

## 11. 生成方法

### 11.1 参考图角色

原始参考图只作为角色身份、画法、配色和情绪参考，不直接裁切其中某一格作为正式素材。

先生成一张无道具、正面自然站立的标准角色母版。母版通过验收后，后续每张状态图都同时引用：

1. 原始角色设定图；
2. 已确认的标准角色母版；
3. 当前状态的独立动作和道具要求。

不在一次大图中生成全部状态。不同状态分别生成和验收，避免同一张角色表中出现比例漂移、裁切困难和透明背景不一致。

### 11.2 通用 Prompt 骨架

```text
Use case: illustration-story
Asset type: 宝宝档案移动端透明角色插图
Primary request: 生成“宝宝饱饱”黄色小鸡的一个状态插图
Input images: 角色设定图作为身份和画法参考；标准角色母版作为身份锁定参考
Subject: 圆润黄色小鸡，橙色爱心头冠，白色波浪边围兜，黄色爱心胸章，深棕圆眼睛，橘粉腮红
Style/medium: 温暖儿童绘本水彩与轻蜡笔质感，柔和轮廓，清楚的小尺寸识别
Composition/framing: 单角色居中，完整轮廓，四周至少 8% 安全边距
Lighting/mood: 柔和暖光，亲切、安定、低压力
Constraints: 保持角色身份锚点、身体比例、颜色和画法一致；透明背景；无地面阴影
Avoid: 文字、数字、水印、复杂背景、额外角色、真实羽毛、强 3D 塑料感、夸张痛苦表情
```

每个状态只追加对应动作、道具和食物质地，不重写角色描述。

## 12. 降级与无障碍

- `<img>` 必须设置能说明当前状态的 `alt`；
- 装饰性母版使用空 `alt`，避免重复朗读标题；
- 图片加载失败时退化为现有 Lucide 图标和文字，不阻断操作；
- 任何业务状态都必须能仅通过文字、选中态和控件属性理解；
- 不以角色颜色作为唯一状态区分；
- 低性能设备使用静态图片，不加载额外动画资源。

## 13. 生成与接入顺序

### 阶段一：概念验证

先生成并接入四张：

1. `chick-master.png`；
2. `age-09-10.webp`；
3. `avoid-none.webp`；
4. `texture-soft-lumps.webp`。

四张图用于验证角色一致性、透明边缘、页面尺寸和小图可读性。

### 阶段二：补齐状态

概念验证通过后，再生成剩余年龄、忌口和质地素材。每完成一类先在真实页面中检查，再进入下一类。

### 阶段三：全流程验收

从年龄、忌口到进食能力完整操作一次，检查预加载、状态切换、滚动、固定底部按钮、低动效模式和图片失败降级。

## 14. 验收标准

### 14.1 角色一致性

- 随机并排任意三张，能够立即判断是同一只小鸡；
- 头冠、围兜、胸章、眼睛和腮红没有漂移；
- 角色主体比例差异只来自年龄状态的计划变化；
- 不出现多余服装、配饰、角色或背景。

### 14.2 状态识别

- 不看文字也能区分忌口的三个回答状态；
- 四种质地在 `56×56px` 下仍有明显区别；
- 年龄形象能够形成成长感，但不会被理解为进食能力等级；
- 风险信息不被角色表情或装饰弱化。

### 14.3 技术与体验

- 切换已缓存素材后，视觉反馈在 `100ms` 内开始；
- 页面切换无明显布局跳动；
- 最后一个字段能够完整滚动到底部操作区上方；
- 单张运行素材不超过 `80KB`；
- 图片失败、离线和 reduced motion 状态均可正常完成档案；
- 类型检查、Lint、生产构建和档案端到端测试全部通过。

## 15. 与现有设计系统的关系

本规范是现有 `设计DNA.json` 的局部扩展：

- 保留奶油白、暖黄、柔粉、低对比描边和大圆角；
- 保留一个页面只有一个主视觉焦点和一个主行动；
- 将“当前不使用角色形象”调整为“角色在主流程中承担向导、解释、陪做和反馈职责”；
- 高风险、高信息密度页面仍以文字、语义颜色和线性图标为主，角色退到辅助位或暂时隐藏；
- 不引入 Canvas、WebGL、粒子、复杂视差或持续动画。

角色不是覆盖所有页面的贴纸，而是有明确任务、视线方向和动作语义的产品参与者。

## 16. 全站角色职责模型

同一只小鸡在不同页面可以换动作，但不能换人格。所有页面先确定角色职责，再选择姿势。

| 职责 | 用户此时需要什么 | 角色行为 | 典型道具 | 禁止行为 |
|---|---|---|---|---|
| 欢迎者 | 知道从哪里开始 | 挥手、自然站立、身体微微朝向主入口 | 无道具或手机 | 同时指向多个入口 |
| 探索向导 | 看懂计划、食物路线或下一站 | 展开地图、抱日历、提篮、指向路线 | 地图、日历、菜篮 | 把计划表达成医学建议 |
| 检查员 | 等待分析或补充事实 | 看手机、拿放大镜、核对夹板 | 手机、放大镜、夹板 | 提前举勾或表现“保证安全” |
| 厨房伙伴 | 跟随当前行动完成制作 | 备料、搅拌、计时、检查质地、端盘 | 厨师帽、勺、锅、计时器 | 接触生熟食材不分、遮挡步骤文字 |
| 记录员 | 回顾一次结果并留下反馈 | 抱记录板、指向单选项、翻日历 | 记录板、日历 | 对“吃得少”“拒绝”做负面评价 |
| 恢复向导 | 从失败、空状态或迷路中恢复 | 举问号牌、看地图、轻指返回方向 | 问号牌、地图 | 哭泣、慌张、故障卖萌 |

### 16.1 情绪状态

- `neutral`：自然微笑，用于默认和未选择状态；
- `focused`：视线看向道具或当前内容，用于分析、检查和记录；
- `confirming`：轻微点头或举勾，用于用户已经完成的普通状态；
- `encouraging`：轻挥手或打开手掌，用于继续下一步；
- `paused`：嘴型收敛、双翅放低，用于暂不建议和无法判断；
- 不制作 `panic`、`crying`、`pain`、`sick` 等状态。

### 17.4 对话陪做

陪做是角色出现最频繁的页面，但必须分成“对话身份”和“当前行动”两层：

1. 每条助手消息左侧的餐叉图标替换为 `36–40px` 的厨师版小鸡头像；
2. 当前行动卡右下角放一张 `80–96px` 的半身动作图，保持文字区域完整；
3. 已完成步骤收起后只保留 `24px` 头像或勾，不重复完整角色；
4. 输入框、快捷选项和固定底栏不放角色，避免压缩操作空间；
5. 出现异常或风险中断时，隐藏当前行动动作图，助手头像切为无厨师道具的 `paused` 状态，风险盾牌成为唯一视觉焦点。

| 当前行动 | 姿势与道具 | 视线与构图 |
|---|---|---|
| 并行准备食材 | 戴厨师帽抱菜篮，旁边露出米碗、番茄和青菜 | 看向食材，角色在卡片右下 |

| 加入青菜 | 端小碗把细碎青菜加入锅中 | 青菜短碎且已焯软，不画长叶或生硬质感 |
| 检查最终质地 | 拿放大镜并用勺背轻压米粒 | 放大质地状态，表情专注，不提前庆祝 |
| 放凉并喂食 | 端成品餐盘并伸翅做“稍等”手势 | 表达先放凉，不画小鸡自己喂宝宝 |

宝宝虾滑面五阶段复用同一动作语法：菜篮备料、搅拌虾泥、小勺下虾滑、计时煮面、端盘放凉。不同菜谱可以换食物和道具，不能改变角色比例和身份锚点。

## 19. 位置、尺寸与密度规则

### 19.1 尺寸层级

| 层级 | CSS 显示尺寸 | 用途 |
|---|---:|---|
| `avatar` | `32–40px` | 对话身份、当前站小标记 |
| `card` | `48–64px` | 选项卡、空状态小提示 |
| `support` | `72–96px` | 计划、结果、反馈页头辅助 |
| `hero` | `104–144px` | 建档、首页、分析、完成页主视觉 |

### 19.2 密度上限

- 单个手机首屏最多出现一张 `hero/support` 角色和一个 `avatar`；
- 同一屏最多两个角色实例，且只能有一个完整身体；
- 长列表不把角色复制到每一行；
- 同一张卡内角色、图标和照片只保留一个主视觉系统；
- 角色不得占用主操作按钮的可点击面积，也不得盖住标题、输入框和风险文案；
- 角色与容器边缘可以轻叠 `8–12px`，但透明安全边距仍需保留。

### 19.3 朝向规则

- 页面右侧角色的视线朝左，看向标题、输入框或主内容；
- 页面左侧角色的视线朝右，看向表单与选项；
- 居中角色看向手中道具，不直视用户制造无意义吸引；
- 翅膀只能指向一个当前主行动；页面有多个同级操作时保持自然姿势。

## 20. 风险覆盖规则

当页面进入风险状态时，按以下优先级覆盖常规角色表现：

1. 风险标题、处理建议和 `ShieldAlert` 必须保持最高对比度；
2. 角色不得出现在风险卡内部；
3. 原本的庆祝、举勾、端餐动作立即停止，切到 `paused` 或隐藏；
4. 不使用红脸、流泪、惊吓、冒汗来表达风险；
5. 不让角色指向“继续制作”“继续喂食”等与风险处理冲突的操作；
6. 紧急医疗提示页面完全隐藏角色，只保留事实、行动与求助路径。

## 21. 实施优先级与验收

### P0：建立产品身份

- 首页 Hero；
- 三步建档；
- 宝宝档案；
- 对话陪做头像。

验收目标：用户在首页、档案和陪做中都能立即识别同一只小鸡，并理解它是产品向导而不是宝宝本人。

### P1：形成完整主流程

- 视频分析四状态；
- 适配结果五结论状态；
- 陪做当前行动姿势；
- 制作完成和即时反馈。

验收目标：角色动作与流程阶段一致；隐藏文字后仍能判断是在“检查、做饭、完成还是记录”，但不能仅凭角色判断安全结论。

### P2：补齐探索和恢复

- 计划、食物地图和食材详情；
- 延迟反馈、历史页；
- 空状态、分析失败和 404。

验收目标：角色增加亲和力和方向感，不降低长列表扫描效率，也不挤压固定底部操作区。

### 全站验收补充

- 任意连续三个页面中的头冠、围兜、胸章、眼睛比例和色彩一致；
- 角色的视线、翅膀和道具都能回答“它在帮助用户做什么”；
- 一屏角色数量符合第 19 节密度上限；
- 风险场景符合第 20 节覆盖规则；
- 图片关闭或加载失败时，全部任务仍能通过文字、语义图标和控件完成；
- `prefers-reduced-motion`、窄屏、长文案和 `200%` 字体缩放下不遮挡操作。

## 22. 动态数据与角色鲁棒性

第 17 节中的具体菜谱姿势是 Mock 场景示例，不是页面判断逻辑。正式实现不得写成“第 1 步用菜篮、第 2 步用搅拌锅”或通过标题包含“青菜”“焖煮”来选择素材。

### 22.1 角色选择优先级

角色由前端确定性映射器选择，优先级从高到低：

1. `risk_interrupted` 或紧急风险：隐藏完整角色，只保留中性 `paused` 头像或完全无角色；
2. 页面固定状态：加载失败、空状态、未知资源使用对应恢复姿势；
3. 经过 Schema 校验的 `visualIntent`；
4. 当前行动的结构化 `actionKind`；
5. 无法识别时使用 `generic` 标准厨师姿势；
6. 通用素材也不可用时不显示图片，保留原有图标和文字布局。

AI 服务只能返回受限枚举，不能返回文件路径、CSS 类名、Prompt 或任意素材 URL。最终素材路径由前端本地映射表决定。

```ts
type CharacterVisualIntent =
  | "welcome"
  | "inspect"
  | "plan"
  | "explore"
  | "prepare"
  | "mix"
  | "heat"
  | "timer"
  | "add"
  | "check_texture"
  | "cool"
  | "serve"
  | "record"
  | "question"
  | "paused"
  | "generic";

function resolveCharacterVisual(input: {
  riskState: "normal" | "caution" | "interrupted";
  visualIntent?: CharacterVisualIntent;
  actionKind?: string;
}): CharacterVisualIntent | "hidden";
```

### 22.2 通用动作映射

| `actionKind` | 首选姿势 | 无专用素材时 | 不能推断的内容 |
|---|---|---|---|
| `prepare`、`cut` | 菜篮备料或砧板旁观察 | `generic` 厨师站姿 | 具体食材、切碎大小 |
| `mix` | 搅拌碗或小锅 | `generic` 厨师站姿 | 搅拌方向、容器材质 |
| `heat`、`simmer` | 看锅并保持距离 | `timer` 或 `generic` | 火力、熟制完成状态 |
| `timer`、`wait` | 拿计时牌或做等待手势 | `generic` | 具体分钟数；数字必须由网页渲染 |
| `add` | 端小碗加入食材 | `generic` | 要加入哪种食材及用量 |
| `check_texture` | 勺背轻压并观察 | `inspect` | “已经适合宝宝”的结论 |
| `cool` | 端盘并做等待手势 | `serve` | 已达到安全入口温度 |
| `serve` | 端成品餐盘 | `generic` | 可以开始喂、食材安全 |
| 未知值 | `generic` | 隐藏角色 | 不通过标题关键词猜测 |

姿势只表达动作类别；食材名称、用量、时间、完成标准和风险结论始终来自结构化数据与页面文字。

### 22.3 数据变化时的同步

- 步骤新增或重排：按稳定 `stepId` 重新解析当前动作，不沿用旧数组位置的图片；
- 菜谱修订：角色、快捷回答、语音和当前行动必须从同一 `recipeRevision` 更新；
- 当前步骤被删除：先隐藏步骤角色并显示暂停状态，直到新快照确认；
- 用户切换菜谱或会话：立即取消旧图片预加载和旧语音，禁止旧角色状态闪入新页面；
- 结果结论变化：从 `question/inspect` 切换到对应姿势，但风险覆盖规则始终优先；
- 宝宝档案变化：年龄形象随新档案更新，菜谱结论与做饭姿势不得仅靠换图假装已经重新适配。

### 22.4 缺失、未知与素材错误

| 情况 | 视觉降级 |
|---|---|
| `visualIntent` 缺失 | 使用页面默认或 `actionKind` 映射 |
| 收到未知枚举 | 记录遥测，使用 `generic`，不崩溃 |
| 图片 `404`、解码失败或超时 | 移除图片占位，显示 Lucide 图标；不保留空白大区域 |
| WebP 不支持 | 使用构建阶段生成的 PNG 备用资源 |
| 慢网尚未加载 | 先渲染固定尺寸容器和低对比占位，不阻塞内容与按钮 |
| `prefers-reduced-motion` | 直接换图，不做淡入缩放 |
| 风险状态与普通姿势冲突 | 风险状态覆盖并隐藏普通姿势 |
| 服务端返回素材 URL | 忽略 URL，只读取受限枚举并走本地映射 |

### 22.5 测试矩阵

角色接入测试不能只覆盖两份 Mock 菜谱，至少生成以下契约样本：

- `0`、`1`、`5`、`12` 个步骤；
- 步骤缺少可选完成标准或计时；
- 未知 `actionKind` 和未知 `visualIntent`；
- 步骤在会话中新增、删除、重排；
- 菜谱修订到达时旧 AI 响应仍在路上；
- 两个不同会话连续切换；
- 图片加载失败、离线和缓存旧版本；
- 正常、注意、风险中断三个安全状态；
- 长菜名、长步骤标题和 `200%` 字体缩放；
- Mock Gateway 与 Remote Gateway 使用同一测试集并得到相同视觉意图。

验收标准：任意合法动态菜谱都能得到正确或中性的角色降级；任何未知、缺失或过期数据都不能导致页面崩溃、串菜谱、显示错误动作，或把未知风险表达成安全确认。
