# 宝宝饱饱：角色视觉规范

> 状态：方案稿
> 适用范围：宝宝档案三步流程及其后续编辑页
> 参考图：`/Users/bytedance/Downloads/ChatGPT Image 2026年7月22日 11_06_09.png`
> 更新日期：2026-07-22

## 1. 目标

以参考图中的黄色小鸡为宝宝饱饱的角色形象，在宝宝档案中承担三类作用：

1. 对用户刚刚选择的年龄、忌口和进食能力给出即时视觉反馈；
2. 降低首次建档的表单感和压力感；
3. 用一致的角色和食物插图帮助用户理解不同食物质地。

角色插图是状态反馈，不是新的输入控件，也不替代文字、校验和无障碍信息。

## 2. 使用边界

### 2.1 首期使用

- `/onboarding/age`：根据实际月龄切换年龄形象；
- `/onboarding/avoid`：根据忌口回答状态切换动作；
- `/onboarding/stage`：展示四种进食阶段及食物质地；
- `/baby/edit`：复用已经生成的档案插图，不新增一套角色。

### 2.2 首期不使用

- AI 分析结论；
- 风险、异常和医疗提示；
- 厨房陪做的当前行动；
- 长列表、设置页和高信息密度页面。

原因：这些页面的第一目标是清楚、可信和快速扫读。角色不能抢占风险信息或当前操作的视觉优先级。

### 2.3 产品边界

- 年龄形象只表达成长感，不表达宝宝理应具备的进食能力；
- 进食能力仍以用户选择的实际表现为准，不根据年龄插图自动推断；
- 纠正月龄不改变基础年龄形象，只在界面中显示纠正月龄标签；
- 插图不构成喂养、发育或医疗建议；
- 风险状态使用现有语义颜色、文字和图标，不把小鸡做成夸张痛苦表情。

## 3. 角色身份

所有状态必须看起来是同一只小鸡，而不是同一风格下的不同角色。

### 3.1 不可变化特征

- 圆润、短胖、近似椭圆的黄色身体；
- 头顶橙色爱心形软冠；
- 白色波浪边围兜；
- 围兜中央黄色爱心胸章；
- 深棕色圆眼睛和高光点；
- 橘粉色圆腮红；
- 小嘴、小短翅膀和短脚；
- 头部与身体之间没有明显细脖子；
- 正面或轻微四分之三视角，不使用强透视。

### 3.2 允许变化特征

- 姿势、手势和视线方向；
- 表情中的微笑、专注、思考和轻微疑问；
- 与当前状态直接相关的道具；
- 随年龄区间轻微变化的身高比例和动作主动程度；
- 厨师帽只用于明确的做饭场景，不作为常驻身份部件。

### 3.3 禁止变化

- 不改变头冠、围兜、胸章和主色；
- 不增加人类头发、鞋、复杂服装或大面积配饰；
- 不出现拟真羽毛、尖锐爪子或真实家禽结构；
- 不出现文字、数字、品牌标志、水印和 UI 按钮；
- 不使用恐怖、疼痛、哭泣或明显医疗化表情；
- 不让道具遮住头冠、眼睛、围兜爱心等身份锚点。

## 4. 视觉 DNA

### 4.1 色彩

| 用途 | 建议色值 | 说明 |
|---|---:|---|
| 身体主色 | `#FFD86A` | 温暖明亮的蛋黄色，避免高荧光 |
| 身体阴影 | `#F1B94C` | 只用于边缘和体积转折 |
| 头冠 | `#F58A3C` | 与参考图一致的橙色强调 |
| 腮红 | `#F29A7E` | 柔和橘粉，不做高饱和红色 |
| 围兜 | `#FFF9EA` | 接近奶油白，保留暖感 |
| 眼睛 | `#3B2717` | 深暖棕，避免纯黑 |
| 线条 | `#C98532` | 低对比暖棕橙轮廓 |
| 页面承载底色 | `#F8F4EC` | 由网页渲染，不进入图片背景 |

### 4.2 材质与画法

- 儿童绘本式暖色水彩与轻蜡笔质感；
- 轮廓柔和但清楚，缩小后仍能识别；
- 体积依赖柔和明暗，不使用硬质 3D 塑料高光；
- 纹理只保留在角色身体、腮红和道具表面；
- 透明背景，不把参考图中的奶油色画布烘焙进素材；
- 不保留地面投影，避免透明图在卡片中出现灰边。

### 4.3 构图

- 角色主体占画布高度约 `78%—84%`；
- 四周保留不少于画布宽度 `8%` 的安全边距；
- 道具与角色形成单一清楚轮廓，不在四周散落装饰物；
- 年龄形象以全身为主；
- 质地形象以半身鸡仔和放大的碗、勺或餐盘为主；
- 小尺寸素材优先确保头冠、眼睛、围兜和食物状态可识别。

## 5. 年龄状态

不为 `4—36` 的每个月生成独立图片，而使用六个连续区间。六个现有快捷月龄会分别命中一个不同状态。

| 区间 | 文件名 | 动作 | 不能表达的含义 |
|---|---|---|---|
| `4—6 月` | `age-04-06.webp` | 体型最小，安静坐姿，双翅放在围兜前 | 不能暗示只会吃泥糊 |
| `7—8 月` | `age-07-08.webp` | 好奇观察，身体轻微前倾 | 不能暗示已经会独坐 |
| `9—10 月` | `age-09-10.webp` | 自然站立，双翅轻扶围兜 | 不能暗示已经会站立 |
| `11—12 月` | `age-11-12.webp` | 开心挥手，另一只翅膀自然下垂 | 不能暗示已经会走路 |
| `13—18 月` | `age-13-18.webp` | 抱着小星星，动作更主动 | 不能暗示认知或语言水平 |
| `19—36 月` | `age-19-36.webp` | 自信的小帮手姿态，双手叉腰或轻挥手 | 不能暗示会独立做饭 |

动作只用于制造视觉差异。年龄段不能用餐具、食物或复杂运动能力区分，以免把年龄和进食能力错误绑定。

### 5.1 映射规则

```ts
function getAgeIllustration(months: number) {
  if (months <= 6) return "age-04-06";
  if (months <= 8) return "age-07-08";
  if (months <= 10) return "age-09-10";
  if (months <= 12) return "age-11-12";
  if (months <= 18) return "age-13-18";
  return "age-19-36";
}
```

输入无效或尚未填写时，显示标准角色母版，不显示某个具体年龄段。

## 6. 忌口状态

| 状态 | 文件名 | 动作与道具 |
|---|---|---|
| 尚未回答 | `avoid-unanswered.webp` | 拿放大镜检查，表情专注但不焦虑 |
| 目前没有 | `avoid-none.webp` | 举绿色勾牌，轻松微笑 |
| 有，需要填写 | `avoid-has-items.webp` | 抱着简单食材清单或夹板，认真确认 |

忌口图只反映回答状态，不把某种具体食材画成“危险食物”。已经选择的具体食材继续由网页标签展示。

## 7. 进食阶段与食物质地

第三步的插图重点是食物状态，不是把四只鸡仔做成能力等级奖章。

| 状态 | 文件名 | 画面要求 |
|---|---|---|
| 刚开始吃辅食 | `texture-puree.webp` | 鸡仔拿小勺，碗里为完全顺滑泥糊；表面连续，没有颗粒 |
| 稠泥和细碎食物 | `texture-thick-puree.webp` | 较稠泥糊能在勺上保持形状，只有极少量细碎点 |
| 压软的小颗粒 | `texture-soft-lumps.webp` | 碗内有明确、柔软的小颗粒；鸡仔用勺背轻压展示可压软 |
| 软手指食物 | `texture-finger-food.webp` | 盘中是容易压软的短条状食物；鸡仔轻拿一条，不出现硬脆食材 |

### 7.1 小尺寸要求

- 每张卡片左侧显示约 `56×56px` 插图；
- 食物区域至少占图片有效面积的 `35%`；
- 缩小到 `48×48px` 时，仍能区分顺滑、稠泥、软颗粒和条状食物；
- 不依赖细小纹理或文字说明完成区分；
- 选中态由网页卡片的暖黄色底色和单选状态表达，图片自身不使用红绿边框。

## 8. 页面接入

### 8.1 年龄页

- 将现有 `52×52px` 问题图标替换为约 `132×132px` 的动态年龄插图；
- 插图放在问题标题上方，作为本页唯一主视觉；
- 月龄变化后立即切换对应图片；
- 纠正月龄只显示文字标签，不更换主图；
- 固定底部主按钮规则保持不变。

### 8.2 忌口页

- 使用约 `104×104px` 的状态插图；
- 未回答、目前没有和有忌口三种状态切换图片；
- 已选食材标签和清空确认逻辑保持不变；
- 插图不能替代“没吃过不等于不能吃”的文字说明。

### 8.3 进食能力页

- 不增加独立大幅头图，避免页面过长；
- 用四张质地插图替换当前选项卡中的数字方块；
- 选项文字、最近进食情况和补充输入保持原逻辑；
- 吞咽费力等风险提示仍使用语义色和盾牌图标，不使用角色图。

## 9. 动效

- 状态切换：旧图淡出，新图淡入并从 `scale(0.97)` 过渡到 `scale(1)`；
- 时长：`220ms`；
- 缓动：`cubic-bezier(0.22, 1, 0.36, 1)`；
- 图片加载完成后才开始切换，避免空白闪烁；
- 年龄页预加载当前和相邻区间素材；
- `prefers-reduced-motion: reduce` 时取消缩放，仅做即时替换；
- 不制作 GIF、视频、逐帧动画、Canvas 或 WebGL 效果。

动效只确认状态变化，不持续吸引注意。

### 10.1 输出规格

- 生成源文件：`1024×1024 PNG`；
- 背景：透明；
- 网页运行素材：透明 `WebP`；
- 年龄和忌口素材：建议输出 `320×320` 与 `640×640` 两档；
- 质地卡片素材：建议输出 `160×160` 与 `320×320` 两档；
- 单张运行素材目标：不超过 `80KB`；
- 所有素材必须使用相同画布、角色比例和安全边距；
- 保留 PNG 母版，WebP 仅作发布版本。

## 11. 生成方法

### 11.1 参考图角色

原始参考图只作为角色身份、画法、配色和情绪参考，不直接裁切其中某一格作为正式素材。

先生成一张无道具、正面自然站立的标准角色母版。母版通过验收后，后续每张状态图都同时引用：

1. 原始角色设定图；
2. 已确认的标准角色母版；
3. 当前状态的独立动作和道具要求。

不在一次大图中生成全部状态。不同状态分别生成和验收，避免同一张角色表中出现比例漂移、裁切困难和透明背景不一致。

### 11.2 通用 Prompt 骨架

```text
Use case: illustration-story
Asset type: 宝宝档案移动端透明角色插图
Primary request: 生成“宝宝饱饱”黄色小鸡的一个状态插图
Input images: 角色设定图作为身份和画法参考；标准角色母版作为身份锁定参考
Subject: 圆润黄色小鸡，橙色爱心头冠，白色波浪边围兜，黄色爱心胸章，深棕圆眼睛，橘粉腮红
Style/medium: 温暖儿童绘本水彩与轻蜡笔质感，柔和轮廓，清楚的小尺寸识别
Composition/framing: 单角色居中，完整轮廓，四周至少 8% 安全边距
Lighting/mood: 柔和暖光，亲切、安定、低压力
Constraints: 保持角色身份锚点、身体比例、颜色和画法一致；透明背景；无地面阴影
Avoid: 文字、数字、水印、复杂背景、额外角色、真实羽毛、强 3D 塑料感、夸张痛苦表情
```

每个状态只追加对应动作、道具和食物质地，不重写角色描述。

## 12. 降级与无障碍

- `<img>` 必须设置能说明当前状态的 `alt`；
- 装饰性母版使用空 `alt`，避免重复朗读标题；
- 图片加载失败时退化为现有 Lucide 图标和文字，不阻断操作；
- 任何业务状态都必须能仅通过文字、选中态和控件属性理解；
- 不以角色颜色作为唯一状态区分；
- 低性能设备使用静态图片，不加载额外动画资源。

## 13. 生成与接入顺序

### 阶段一：概念验证

先生成并接入四张：

1. `chick-master.png`；
2. `age-09-10.webp`；
3. `avoid-none.webp`；
4. `texture-soft-lumps.webp`。

四张图用于验证角色一致性、透明边缘、页面尺寸和小图可读性。

### 阶段二：补齐状态

概念验证通过后，再生成剩余年龄、忌口和质地素材。每完成一类先在真实页面中检查，再进入下一类。

### 阶段三：全流程验收

从年龄、忌口到进食能力完整操作一次，检查预加载、状态切换、滚动、固定底部按钮、低动效模式和图片失败降级。

## 14. 验收标准

### 14.1 角色一致性

- 随机并排任意三张，能够立即判断是同一只小鸡；
- 头冠、围兜、胸章、眼睛和腮红没有漂移；
- 角色主体比例差异只来自年龄状态的计划变化；
- 不出现多余服装、配饰、角色或背景。

### 14.2 状态识别

- 不看文字也能区分忌口的三个回答状态；
- 四种质地在 `56×56px` 下仍有明显区别；
- 年龄形象能够形成成长感，但不会被理解为进食能力等级；
- 风险信息不被角色表情或装饰弱化。

### 14.3 技术与体验

- 切换已缓存素材后，视觉反馈在 `100ms` 内开始；
- 页面切换无明显布局跳动；
- 最后一个字段能够完整滚动到底部操作区上方；
- 单张运行素材不超过 `80KB`；
- 图片失败、离线和 reduced motion 状态均可正常完成档案；
- 类型检查、Lint、生产构建和档案端到端测试全部通过。

## 15. 与现有设计系统的关系

本规范是现有 `设计DNA.json` 的局部扩展：

- 保留奶油白、暖黄、柔粉、低对比描边和大圆角；
- 保留一个页面只有一个主视觉焦点和一个主行动；
- 将“当前不使用角色形象”调整为“角色只用于低压力、情感型和成长型页面”；
- 高风险、高信息密度和操作型页面继续使用文字、语义颜色和线性图标；
- 不引入 Canvas、WebGL、粒子、复杂视差或持续动画。

角色是产品信息层级的辅助，不是覆盖所有页面的装饰层。
